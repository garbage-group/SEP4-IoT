 This repository contains a collection of drivers designed to work with the hardware at SEP4 at Software Technology Engineering at VIA. All drivers are located in the lib/drivers directory.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysubsection{\texorpdfstring{1. adxl345.\+h}{1. adxl345.h}}\label{md__r_e_a_d_m_e_autotoc_md1}
This driver interfaces with the ADXL345 chip, a 3-\/axis accelerometer, that measures up to Â±4 g. It can be used to measure the static acceleration of gravity in tilt-\/sensing applications, as well as dynamic acceleration resulting from motion or shock.\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{\texorpdfstring{2. buttons.\+h}{2. buttons.h}}\label{md__r_e_a_d_m_e_autotoc_md2}
This driver is designed to control the three buttons located on top of the multifunction shield.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{\texorpdfstring{3. buzzer.\+h}{3. buzzer.h}}\label{md__r_e_a_d_m_e_autotoc_md3}
Controls the active buzzer positioned on the multifunction shield.\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsection{\texorpdfstring{4. dht11.\+h}{4. dht11.h}}\label{md__r_e_a_d_m_e_autotoc_md4}
The DHT11 is a basic digital temperature and humidity sensor. This driver allows for interfacing with the DHT11 to obtain temperature and humidity readings.\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{\texorpdfstring{5. display.\+h}{5. display.h}}\label{md__r_e_a_d_m_e_autotoc_md5}
This driver controls the four 7-\/segment displays positioned on top of the multifunction shield. It can be used to display numbers or certain characters.\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{\texorpdfstring{6. hc\+\_\+sr04.\+h}{6. hc\_sr04.h}}\label{md__r_e_a_d_m_e_autotoc_md6}
The HC-\/\+SR04 is an ultrasonic ranging module that provides 2 cm to 400 cm non-\/contact measurement functionality. This driver facilitates the measurement of distance using ultrasonic waves.\hypertarget{md__r_e_a_d_m_e_autotoc_md7}{}\doxysubsection{\texorpdfstring{7. leds.\+h}{7. leds.h}}\label{md__r_e_a_d_m_e_autotoc_md7}
Controls the four LEDs located on top of the multifunction shield. This can be used for indication, signaling, or other visual purposes.\hypertarget{md__r_e_a_d_m_e_autotoc_md8}{}\doxysubsection{\texorpdfstring{8. light.\+h}{8. light.h}}\label{md__r_e_a_d_m_e_autotoc_md8}
This driver controls the photoresistor and is capable of measuring the amount of light in the environment.\hypertarget{md__r_e_a_d_m_e_autotoc_md9}{}\doxysubsection{\texorpdfstring{9. pc\+\_\+comm.\+h}{9. pc\_comm.h}}\label{md__r_e_a_d_m_e_autotoc_md9}
Facilitates communication to the PC over USB using UART.\hypertarget{md__r_e_a_d_m_e_autotoc_md10}{}\doxysubsection{\texorpdfstring{10. periodic\+\_\+task.\+h}{10. periodic\_task.h}}\label{md__r_e_a_d_m_e_autotoc_md10}
This driver sets up an interrupt-\/based service that can call a function at a specified interval. It\textquotesingle{}s useful for tasks that need to be executed periodically without manual intervention.\hypertarget{md__r_e_a_d_m_e_autotoc_md11}{}\doxysubsection{\texorpdfstring{11. pir.\+h}{11. pir.h}}\label{md__r_e_a_d_m_e_autotoc_md11}
This driver interfaces with the HC-\/\+SR501 is a passive infrared (PIR) motion sensor.\hypertarget{md__r_e_a_d_m_e_autotoc_md12}{}\doxysubsection{\texorpdfstring{12. servo.\+h}{12. servo.h}}\label{md__r_e_a_d_m_e_autotoc_md12}
This driver interfaces with the SG90 servo motor and can set it to a position ranging from 0 to 180 degrees.\hypertarget{md__r_e_a_d_m_e_autotoc_md13}{}\doxysubsection{\texorpdfstring{13. tone.\+h}{13. tone.h}}\label{md__r_e_a_d_m_e_autotoc_md13}
This driver can generate tones of specified frequencies, using a passive buzzer. By providing different frequency values, you can play various tones or even simple melodies.\hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysubsection{\texorpdfstring{14. uart.\+h}{14. uart.h}}\label{md__r_e_a_d_m_e_autotoc_md14}
This driver facilitates UART (Universal Asynchronous Receiver-\/\+Transmitter) communication. It is utilized by both the pc\+\_\+comm and wifi drivers to enable data transfer and communication functionalities.\hypertarget{md__r_e_a_d_m_e_autotoc_md15}{}\doxysubsection{\texorpdfstring{15. wifi.\+h}{15. wifi.h}}\label{md__r_e_a_d_m_e_autotoc_md15}
This driver interfaces with the ESP8266 module, a Wi-\/\+Fi microchip with a full TCP/\+IP stack and microcontroller capability. With this driver, you can connect to an Access Point (AP) and establish a TCP connection, and send and receive data through the TCP-\/connection.
